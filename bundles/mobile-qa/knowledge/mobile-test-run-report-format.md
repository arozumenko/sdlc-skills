# Mobile Test Run Report Format Specification

## File Location & Naming

```
reports/
  RUN-YYYY-MM-DD-NNN.md     ← main report
  screenshots/
    TC-001_YYYY-MM-DD.png
    TC-002_YYYY-MM-DD.png
    ...
  manual-guides/
    TC-NNN-guide.md          ← step-by-step guide for native manual runs
```

Run ID format: `RUN-2026-05-18-001`. Sequence number resets per day.

---

## Section Reference

| Section | Required | Description |
|---------|----------|-------------|
| YAML frontmatter | Yes | Machine-readable metadata; `environment` only for PWA/hybrid |
| Summary table | Yes | Aggregate metrics |
| Runner Mode Breakdown | Yes | Manual vs Playwright split |
| Size Distribution | If sizes present | S/M/L breakdown; omit when all `size` are null |
| Results table | Yes | One row per TC, sorted by ID; includes Platform, Mode columns |
| Failed Tests | If failures | Detailed breakdown with screenshot path and platform observed |
| Blocked Tests | If blocked | Short explanation with remediation hint |
| Defects Found | If failures | Checklist-style; one item per unique failure |
| Manual Execution Guides | If manual cases | List of guide paths generated by runner |
| Notes | Yes | Caveat about autonomous vs manual execution |

---

## Status Definitions

| Status | Icon | Meaning |
|--------|------|---------|
| `PASS` | ✅ | All steps executed, all expected results confirmed |
| `FAIL` | ❌ | At least one step produced unexpected result |
| `BLOCKED` | ⏸ | Could not execute — precondition unmet or environment issue |

## Executor Result JSON Schema

Each `mobile-test-runner` agent outputs one JSON block. The `mobile-test-reporter` consumes an array of these.

```json
{
  "tc_id": "TC-001",
  "title": "Login with valid credentials",
  "priority": "critical",
  "platform": "both",
  "app_type": "native",
  "runner_mode": "manual",
  "size": "S",
  "result": "PASS",
  "steps_total": 5,
  "steps_completed": 5,
  "failure_step": null,
  "failure_reason": null,
  "screenshot": "reports/screenshots/TC-001_2026-05-18.png",
  "manual_guide": null,
  "duration_seconds": 130,
  "notes": "",
  "tokens": null,
  "tool_uses": null,
  "duration_ms": null
}
```

`manual_guide`: path to the generated guide file when `runner_mode` is `manual` and result is `BLOCKED`; `null` otherwise.  
`tokens`, `tool_uses`, `duration_ms`: attached by `mobile-run-lead` from usage reporting; may be null (Performance Metrics section omitted when null).
